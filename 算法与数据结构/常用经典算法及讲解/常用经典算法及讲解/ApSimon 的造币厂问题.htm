ApSimon 的造币厂问题

摘要    本文介绍了作者在艾波西蒙造币厂问题上的研究进展: (1) 给出了n=6,7 时的更好的上界: min(P6)≤32, min(p7)≤64,当n=6时,ai,bi依次取为(8,6,6,0,4,1),

(0,3,6,5,5,2),当n=7时,ai,bi依次取为(11,11,10,3,8,8,0),(0,2,2,9,9,8,6). (2) 给出了一种解决艾波西蒙问题的计算机搜索算法,对于给定的n,按此算法可找出较好的pn.这是迄今所见到的关于该问题的最好结果.

1  问题简介及主要结果

 

    艾波西蒙(ApSimon)于1984年在他所写的一本书[1]中,介绍了下面的造币厂问题:

     n 个造币厂生产同一种硬币, 但其中某些厂由于材料问题造出了非标准的硬币.设标准的硬币重c克(已知),非标准的硬币只有一种重量,重c(1+e)克,其中e是个不为0的未知数,可以取正数或负数.为了查出哪些厂生产的硬币是非标准的,从各厂中抽出一些样品(个数不限)放在一起进行称重,只能称2次.设ai,bi(i=1,...,n)分别为第一次,第二次称重从第i个厂中取得的样品个数.每个厂提供的样品或者都是标准的,或者都是非标准的.设pn=∑max(ai,bi)为某一称重方案的样品总数,求一种使pn最小的称重方案(参看[3]).

    其中∑表示对于i从1到n求和,即∑i=1,…,n(以下同).

    艾波西蒙给出了以下结果:

    (1) 若取 bi=1, ai=i!,(i=1,...,n),则问题必有解.

    (2) 对于n=3,4,5,有:

      min(p3)=4,ai取为(0,1,2), bi取为(1,1,0).

      min(p4)=8,ai取为(0,1,1,4),bi取为(2,0,1,1).

      min(p5)=15,ai取为(1,0,1,4,5),bi取为(1,2,2,5,0).

根据上述结果,艾波西蒙提出了2个问题:

   A.1  是否对每一个n,均有min(pn)≤2n-1 ?

   A.2  min(pn)=?

    由于只经过2次称重,便可在n个厂中找出所有生产非标准硬币的工厂,故这一问题在理论上和实际应用中(例如测试技术)都具有较大的意义.但该问题迄今仍未解决.1994年,在著名的《The American Mathematical Monthly》的"Unsolved Problems"栏目,R.Guy[2]给出了用计算机得到的以下结果:

   min(p6)≤38,ai取为(1,1,2,3,5,8),bi取为(11,6,5,3,1,1);

   min(p7)≤74,ai取为(0,1,3,6,10,15,21),bi取为(1,5,8,14,10,8,0).

但这里的 38,74 只是个上界,不是上确界, 由此不能否定问题 A.1.

   本文介绍了作者在艾波西蒙问题上的研究进展:

   L.1  给出了n=6,7 时的更好的上界: min(P6)≤32, min(p7)≤64,当n=6时,ai,bi依次取为(8,6,6,0,4,1),(0,3,6,5,5,2),当n=7时,ai,bi依次取为(11,11,10,3,8,8,0),(0,2,2,9,9,8,6).这就关于n=6,7 对问题A.1给予了肯定的回答.

   L.2  给出了一种解决问题A.2的计算机搜索算法,对于给定的n,按此算法可找出较好的pn, 进而可找出min(pn).

   这是目前在国内外文献上所见到的最好的结果.

 

2 有关的分析

 

令di(i=1,...,n)为对第i个厂检测的结果,若di=0,表示第i个厂生产的硬币是标准的,di=1,表示第i个厂生产的硬币是非标准的.U1,U2为两次称重的结果,则有:  U1=∑c(1+die)ai, U2=∑c(1+die)bi, 由于 U1,U2,c为已知,令W1=U1/c,W2=U2/c,则有 W1=∑(1+die)ai, W2=∑(1+die)bi, ∑diai=(W1-∑ai)/e, ∑dibi=(W2-∑bi)/e,  令  R=

e∑diai=W1-∑ai, 令S=e∑dibi=W2-∑bi.

对于n个造币厂,每个di均有2种可能(取0或1),故(d1,d2,...,dn)共有2n种不同的组合,如果R/S=(∑diai)/(∑dibi)恰好有2n个不同的比值,便可和(d1,d2,...,dn)建立一一对应的关系,从而由R/S的值便可断定哪些厂生产的硬币是非标准的.于是问题转化为求(a1,a2,...,an),(b1,b2,...,bn),使得对于(d1,d2,...,dn)的任何两组不同的值,均可得到R/S 的两个不同的值.

 

3  算法简介

 

  变量简介: 

   samp:钱币样本总数(作为已知数据输入),

   n:工厂数(已知),

   a[i],b[i]:i=1,...,n,分别为第一次,第二次称重从第i个厂中取得的样品个数(作为结果输出).

   m=[n/2],当i≤m时,a[i]≥b[i],当i>m时,a[i]≤b[i],并且,a[1]≥a[2]≥...≥a[m]≥b[m+1]≥...≥b[n], 显然,应有:samp=a[1]+a[2]+...+a[m]+b[m+1]+...+b[n].

   ss:记录已使用的样品总数.

   算法 A.对于给定的a[i],b[i],及(d[1],d[2],...,d[n])的所有可能的取值,判断是否能得到2n个不同的R/S=(∑d[i]*a[i])/(∑d[i]*b[i])的值.

    A-1 for i:=1 to n do d[i]:=0;

        t:=1;  d[1]:=1;  计算rs1=R/S;  rj:=1;  e[1]:=rs1;

    A-2 while t<2^n do

       【 t:=t+1; t1:=t;

          for i:=1 to mint do

            【 d[i]:=t1 mod 2;  t1:=t1 div 2 】;

            计算rs1;

          for j:=1 to rj do

            if e[j]=mn1 then  显示"没通过检验",结束;

          inc(rj); e[rj]:=rs1;

       】;

    A-3 通过检验,打印结果.

   算法 B.利用递归生成所有可能的(a[1],a[2],...,a[n]),(b[1],b[2],...,b[n]).

从中搜索满足条件的解.记第i1次递归过程为next(i1).

   B-1 if i1=n then {递归结束控制}

       【b[i1]:=samp-ss;

          if bb[i1]>0

            then  for k:=0 to b[i1] do 【 a[i1]:=k; 执行算法A】

            else 返回上层过程

       】

   B-2 i2:=mint-i1+1;

       c[i1]:=(samp-ss+i2-1) div i2;

       maxt:=c[i1]*2; {c[i1],maxt为对a[i1]或b[i1]搜索的下界,上界}

       if i1>1 then

        【maxt:=a[i1-1];

          if b[i1-1]>a[i1-1] then maxt:=b[i1-1]】;

   B-3 if i1<=(mint div 2) then

         for a[i1]:=c[i1] to maxt do

          【ss:=ss+a[i1];

            if ss<samp then

              for b[i1]:=0 to a[i1] do next(i1+1);

            ss:=ss-aa[i1];

          】

       else

         for b[i1]:=cc[i1] to maxt do

          【ss:=ss+bb[i1];

            if ss<samp then

              for a[i1]:=0 to b[i1] do next(i1+1)

            ss:=ss-bb[i1];

          】   

   主算法:

    (1) 输入 n, samp, 令 ss:=0.

    (2) i1=1, 执行算法 B.(next(i1))

    第 1 段中的结果 L.1 就是利用该算法得到的.计算机程序是用Turbo Pascal 7.0编写的.实际计算表明,当n=6,7时,对固定的样本数,若有解,则解不唯一.容易分析,该算法的时间复杂性是指数型的.为了找到更好的结果(满足min(pn)<2n-1),作者曾在一台奔腾200MHz的微机上运行数小时,鉴于时间太长，没有完成搜索.

 

4 结束语

 

   关于艾波西蒙问题还有许多工作可做.今后力图找出某些内在的规律,进而简化算法,从而可对更大的n进行搜索或得到更小的pn.作者深刻体会到,一些难题在数学理论上的研究,一旦与计算机算法联系起来,与实际的上机计算联系起来,必将有所突破,有所进展.
